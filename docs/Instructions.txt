OpenRouter Signup Automation — Detailed Instructions
=====================================================

Script: openrouter_signup.py
Credentials output: openrouter_credentials.txt

ARCHITECTURE
------------
- Steps 2/5/6 (OpenRouter signup, verify, key extraction): Camoufox (anti-fingerprint Firefox)
  - Uses fresh tmpdir (tempfile.mkdtemp) per run, cleaned up on exit
  - persistent_context=True with user_data_dir=tmpdir
- Step 3 (Proton Mail verification link): Playwright Chromium subprocess
  - Uses persistent profile at ~/proton_profile (survives across runs)
  - If profile already has Proton logged in, skips credential entry
  - Runs in subprocess to avoid Camoufox/Playwright Firefox event-loop crash
    (coreBundle.js bug: pageError.location is undefined on Proton's JS errors)

PREREQUISITES
-------------
1. camoufox + playwright installed: pip install camoufox playwright
2. email.sh present at /home/runner/workspace/email.sh
   - Generates Duck email via ~/duckmail.py (uses Camoufox)
   - Prints email to stdout (captured by subprocess)
   - Also appends to mail.txt
   - Creates ~/config.py with PROTON_USERNAME, PROTON_PASSWORD, credential pools
3. Chromium at: /nix/store/qa9cnw4v5xkxyip6mb9kxqfq1z4x2dx1-chromium-138.0.7204.100/bin/chromium
4. Proton profile dir ~/proton_profile (auto-created)

STEP-BY-STEP FLOW
------------------

STEP 1: Generate Duck Email
  - Runs: bash email.sh
  - Captures email address from stdout (last line with @)
  - Subprocess env includes:
    - PATH: venv bin dir prepended (so python3 resolves to venv)
    - VIRTUAL_ENV: set to venv dir
    - PYTHONPATH: includes .pythonlibs for camoufox access
  - Imports ~/config.py fresh (deletes stale pyc cache, reloads module)
  - Generates random 15-char password (letters + digits + !@#$%)

STEP 2: Sign Up on OpenRouter (Camoufox)
  - Navigates to https://openrouter.ai/sign-up
  - Waits for #emailAddress-field to be visible
  - Fills: email, password, checks Terms of Service checkbox (#legalAccepted-field)
  - Clicks "Continue" button
  - Waits 15s, then checks for Cloudflare challenge frame
    - If found, clicks #challenge-stage / .ctp-checkbox
  - Waits 8s, checks for confirm-email/verification indicator in URL or body

STEP 3: Check Proton Mail for Verification Link (Chromium subprocess)
  - Runs as subprocess (avoids Camoufox Firefox crash)
  - Uses Playwright Chromium with launch_persistent_context at ~/proton_profile
  - Login check: if "Mail" link visible, skips credentials (already logged in)
  - Otherwise fills #username and #password, submits
  - Clicks "Mail" link with timeout=0
  - Inbox search (5 retries, 5s wait each):
    - Presses "/" keyboard shortcut to open Proton search
    - Types signup email address with keyboard.type()
    - Presses Enter to search
    - Clicks first .item-container result
    - If not found, presses Escape and retries
  - Extracts verification link from email frames:
    - Primary: regex for https://clerk.openrouter.ai/v1/verify...
    - Fallback: regex for https://openrouter.ai...verify/confirm/token...
    - Decodes &amp; → &
    - Last fallback: searches anchor tags for verify/confirm + openrouter
  - Outputs: VERIFY_URL:<url> or VERIFY_URL:NOT_FOUND

STEP 4: Verify Email + Sign In (Camoufox)
  - Navigates to verify_url from Step 3
  - If on openrouter.ai, clicks "Individual" (personal account selection)
  - Waits for login to settle (8s)
  - Verification link typically auto-logs in

STEP 5: Extract API Key (Camoufox)
  - Navigates to https://openrouter.ai/workspaces/default/keys
  - If "Create Key" button visible, clicks it to create a new key first
  - Key extraction (3 methods, tried in order):
    1. <code> block: page.locator("code").inner_text() + regex sk-or-v1-[a-zA-Z0-9]+
    2. Copy button + clipboard: click "Copy" button, read navigator.clipboard.readText() + regex
    3. HTML regex: scan full page HTML for sk-or-v1-[a-zA-Z0-9]{20,}

STEP 6: Save Credentials
  - Appends to openrouter_credentials.txt:
    EMAIL=<email>
    PASSWORD=<password>
    API_KEY=<key or NOT_FOUND>

RETRY LOGIC
-----------
- Main loop: 3 attempts
- If Step 3 returns NOT_FOUND (verification email not found after 5 inbox retries):
  - Generates fresh email via email.sh
  - Generates new password
  - Restarts from Step 2
- After 3 failed attempts, prints "FAILED: 3 signup attempts exhausted."

NOTES
-----
- The 6-digit sign-in verification code (factor-two) is NOT handled automatically
  — OpenRouter sends this on sign-in and must be entered manually if prompted
- Playwright coreBundle.js was patched to guard pageError.location before accessing .url
  (fixes Firefox engine crash on Proton Mail JS errors)
- email.sh subprocess needs venv PATH + VIRTUAL_ENV + PYTHONPATH set correctly
  because duckmail.py uses camoufox which is in .pythonlibs, not system python
