CloakBrowser Automation Scripts — General Flow & Methods
=========================================================

All scripts use CloakBrowser (anti-detect Chromium) instead of raw Playwright.
headless=False always (headless=True crashes CloakBrowser on this system).
No xvfb-run needed — display server is running (DISPLAY=:1).

PREREQUISITES
-------------
1. cloakbrowser installed: pip install cloakbrowser
2. email.sh present at scripts/email.sh
   - Generates Duck Address email via ~/duckmail.py (CloakBrowser)
   - Prints email to stdout (captured by calling scripts)
   - Appends to credentials/mail.txt
   - Creates ~/config.py with PROTON_USERNAME, PROTON_PASSWORD, credential pools
3. Proton profile dir ~/proton_profile (auto-created, persists login across runs)
4. Python packages in .pythonlibs (auto-resolved via venv)

RUN ANY SCRIPT WITH:
  bash scripts/SCRIPT_NAME.sh
(No xvfb-run needed — display server is running)

=================================================================
GENERAL AUTOMATION FLOW (3-STEP PATTERN)
=================================================================

All signup/verification scripts follow this pattern:

  STEP 1: SIGNUP (fc_signup.py / or_signup.py)
  - Opens fresh tmpdir browser profile (CloakBrowser, headless=False, humanize=True)
  - Navigates to target signup URL
  - Fills email + generated password
  - Handles site-specific quirks (Turnstile, Clerk checkboxes, etc.)
  - Submits form, waits for verification-email-sent state
  - Prints resulting URL for debugging

  STEP 2: FETCH VERIFICATION LINK (fc_proton.py / or_proton.py)
  - Opens persistent Proton profile at ~/proton_profile (headless=False)
  - Navigates to Proton inbox (u/0 or u/1)
  - Logs in if needed (credentials from ~/config.py)
  - Search loop (15 attempts, ~5s each):
    - Press "/" to open search
    - Type signup email, press Enter
    - Wait for results
    - Iterate up to 5 emails (skip already-checked by subject)
    - Click email, scan frames + links for verify URL matching target domain
    - If found: print VERIFY_URL:<url> and exit
    - If not found: clear checked set, reload inbox
  - If loop exhausts: print VERIFY_URL:NOT_FOUND

  STEP 3: VERIFY + EXTRACT CREDENTIAL (fc_verify.py / or_verify.py)
  - Opens fresh tmpdir browser profile
  - Navigates to verify_url (PKCE token link)
  - If new tab opened by redirect, switches to it
  - Polls for page to land on target domain (40 iterations, 2s each)
  - Handles intermediate states:
    - If lands on /signin or /login: fills credentials, submits, continues polling
    - If link expired (Clerk status=expired): exits with code 2
  - Once on dashboard: navigates to API keys page
  - API key extraction (multiple strategies, tried in order):
    1. Click reveal button (eye icon), read revealed text
    2. Click copy button, read clipboard
    3. Scan page HTML/relevant elements via regex
  - Saves to credentials/<service>_credentials.txt (EMAIL, PASSWORD, API_KEY)

RETRY LOGIC (Main Bash Loop)
----------------------------
- 3 attempts maximum
- On failure at any step: generate fresh email + password, restart from Step 1
- OpenRouter special case: verification link expiry (exit code 2) triggers immediate retry
- After 3 failed attempts: prints "FAILED: 3 attempts exhausted"

SERVICE-SPECIFIC EXAMPLES
=================================================================

--- OpenRouter (openrouter_signup.sh) ---
Signup URL: https://openrouter.ai/sign-up
Quirks:
  - Clerk form: checkbox must be toggled via React fiber onChange (direct click fails)
  - Turnstile: Cloudflare iframe at ~(478,189) size 300x65, NO visible checkbox
    - Only CDP-level mouse.click(frame_x+30, frame_y+h/2) works
    - xdotool & frame.locator.click() FAIL
  - Verification link: clerk.openrouter.ai/v1/verify?...
  - Dashboard: https://openrouter.ai/keys (select "Individual" on first visit)
  - API key format: sk-or-v1-... or sk-... (30+ chars)

--- Firecrawl (firecrawl_signup.sh) ---
Signup URL: https://www.firecrawl.dev/signin → click "Sign Up" tab
Quirks:
  - Standard email/password form (no Clerk/Turnstile)
  - Verification link: contains "firecrawl" + "verify"
  - Dashboard: https://www.firecrawl.dev/app → /api-keys
  - API key format: fc-[a-zA-Z0-9]{20,}
  - Key reveal: lucide-eye / lucide-eye-off icons, or copy button

=================================================================
CLOAKBROWSER PATTERNS (Canonical)
=================================================================

Launch (always use this):
  from cloakbrowser import launch, launch_persistent_context
  ctx = launch_persistent_context(tmpdir, headless=False, humanize=True)
  p = ctx.pages[0] if ctx.pages else ctx.new_page()

Cleanup:
  import atexit, shutil
  td = tempfile.mkdtemp()
  atexit.register(lambda: shutil.rmtree(td, ignore_errors=True))

Key differences from Playwright:
- No executable_path needed (bundled Chromium v146)
- humanize=True for natural mouse/anti-detect timing
- headless=True CRASHES on this system — always use headless=False
- No xvfb-run needed (display server running)
- Stealth is built into the binary (no JS injection needed)

Common Helpers:
- p.wait_for_timeout(ms) — prefer over time.sleep for page interactions
- p.evaluate(js) — run JS in page context (React fiber, clipboard, etc.)
- p.mouse.click(x, y) — CDP-level click (bypasses frame restrictions)
- frame = p.frames[i] — access iframes (Turnstile, Proton message frames)
- frame.frame_element().bounding_box() — get iframe screen coords for mouse.click

=================================================================
EMAIL GENERATION (email.sh)
=================================================================
Purpose: Generate a Duck Address email via Duck Login API
Output: prints email to stdout, appends to credentials/mail.txt

Flow:
  - Checks if cloakbrowser is installed, installs if not
  - Generates ~/config.py with credential pools (Duck + Proton + Firecrawl)
  - Generates ~/duckmail.py (CloakBrowser script)
  - Runs ~/duckmail.py:
    - Opens Duck login page (ddgeep.vercel.app/login)
    - Fills Duck username + access token (random pair from config)
    - Clicks "Generate Private Duck Address"
    - Reads generated email from page
    - Prints email to stdout
  - Appends email to credentials/mail.txt

config.py structure:
  PROTON_USERNAME, PROTON_PASSWORD
  DUCK_CREDENTIALS = [{"username": "...", "access_token": "..."}, ...]
  FIRECRAWL_CREDENTIALS = []  # populated after successful signup

=================================================================
TROUBLESHOOTING
=================================================================

"Missing X server or $DISPLAY":
  - Ensure DISPLAY is exported: export DISPLAY=:1
  - All scripts set this automatically

"TargetClosedError":
  - Page/tab was closed (common with PKCE verify links)
  - Scripts now catch this and check for new tabs (ctx.pages[-1])

Proton not finding email:
  - Email may take 30-60s to arrive
  - Script retries 15 times with 5s waits + inbox reloads
  - Check ~/proton_profile is persisting login (first run needs valid creds)

Checkbox not registering (Clerk/OpenRouter):
  - Use .check(force=True) instead of .click()
  - JS dispatchEvent doesn't update React/Clerk internal state
  - Current solution: evaluate() to find React fiber and call onChange directly

Turnstile not solving:
  - iframe has NO visible checkbox — standard solvers fail
  - Must use CDP mouse.click at frame center (x+30, y+h/2)
  - Wait 15s after click for challenge completion
