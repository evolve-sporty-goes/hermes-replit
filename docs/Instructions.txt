CloakBrowser Automation Scripts — Detailed Instructions
======================================================

All scripts use CloakBrowser (anti-detect Chromium) instead of raw Playwright.
headless=False always (headless=True crashes CloakBrowser on this system).
No xvfb-run needed — display server is running (DISPLAY=:1).

PREREQUISITES
-------------
1. cloakbrowser installed: pip install cloakbrowser
2. email.sh present at scripts/email.sh
   - Generates Duck Address email via ~/duckmail.py (CloakBrowser)
   - Prints email to stdout (captured by calling scripts)
   - Appends to credentials/mail.txt
   - Creates ~/config.py with PROTON_USERNAME, PROTON_PASSWORD, credential pools
3. Proton profile dir ~/proton_profile (auto-created, persists login across runs)
4. Python packages in .pythonlibs (auto-resolved via venv)

RUN ANY SCRIPT WITH:
  bash scripts/SCRIPT_NAME.sh
(No xvfb-run needed — display server is running)

=================================================================
SCRIPT: firecrawl_signup.sh
=================================================================
Purpose: Firecrawl account signup + email verification + API key extraction
Output: credentials/firecrawl_credentials.txt

Architecture:
- Generates ~/fc_signup.py, ~/fc_proton.py, ~/fc_verify.py at runtime
  (same pattern as email.sh generating ~/duckmail.py)
- Each .py helper is a standalone CloakBrowser script called via python3
- Main bash orchestrates the flow with retry logic

STEP-BY-STEP FLOW
-----------------

STEP 1: Generate Duck Email + Load Config
  - Runs: bash scripts/email.sh
  - Captures email from stdout (last line, whitespace stripped)
  - Loads PROTON_USER, PROTON_PASS from ~/config.py via importlib
  - Generates random 16-char password (letters + digits + !@#$%)

STEP 2: Sign Up on Firecrawl (fc_signup.py)
  - Opens fresh tmpdir browser profile (CloakBrowser, headless=False, humanize=True)
  - Navigates to https://www.firecrawl.dev/signin
  - Clicks "Sign Up" tab
  - Fills email + password
  - Clicks "Create Account"
  - Waits 10s, prints resulting URL
  - Expects redirect to /signin/confirm-email

STEP 3: Check Proton Mail for Verification Link (fc_proton.py)
  - Opens persistent profile at ~/proton_profile (headless=False)
  - Navigates to https://mail.proton.me/u/0/inbox
  - If redirected to /login, fills credentials and submits
  - Search loop (up to 15 attempts):
    - Waits 5s for inbox to load
    - Presses "/" to open Proton search
    - Types signup email, presses Enter
    - Waits 5s for results
    - If no results: navigates to inbox URL again (page.goto, not reload)
    - If results found:
      - Iterates through up to 5 emails (skips already-checked by subject)
      - Clicks each email, searches frames + links for verify URL
      - If verify link found: prints VERIFY_URL:<url> and exits
      - If not found in any: clears checked set, navigates to inbox again
  - If loop exhausts: prints VERIFY_URL:NOT_FOUND

STEP 4: Verify Email + Extract API Key (fc_verify.py)
  - Opens fresh tmpdir browser profile
  - Navigates to verify_url (PKCE token link)
  - If new tab opened by redirect, switches to it
  - Polls for page to land on firecrawl.dev (up to 40 iterations, 2s each)
  - If lands on /signin or /login:
    - Fills email + password, clicks "Sign In"
    - Continues polling
  - Once on dashboard: navigates to /api-keys
  - API key extraction (3 methods, tried in order):
    1. Click eye icon (button:has(.lucide-eye)), read revealed text
    2. Click eye-off icon (button:has(.lucide-eye-off))
    3. Click copy button + read clipboard
    4. HTML regex: scan full page HTML for fc-[a-zA-Z0-9]{20,}
  - Saves to credentials/firecrawl_credentials.txt

RETRY LOGIC
-----------
- Main loop: 3 attempts
- If Step 3 returns NOT_FOUND:
  - Generates fresh email + password
  - Restarts from Step 2
- After 3 failed attempts: prints "FAILED: 3 attempts exhausted"

=================================================================
SCRIPT: email.sh
=================================================================
Purpose: Generate a Duck Address email via Duck Login API
Output: prints email to stdout, appends to credentials/mail.txt

Flow:
  - Checks if cloakbrowser is installed, installs if not
  - Generates ~/config.py with credential pools (Duck + Proton + Firecrawl)
  - Generates ~/duckmail.py (CloakBrowser script)
  - Runs ~/duckmail.py:
    - Opens Duck login page (ddgeep.vercel.app/login)
    - Fills Duck username + access token (random pair from config)
    - Clicks "Generate Private Duck Address"
    - Reads generated email from page
    - Prints email to stdout
  - Appends email to credentials/mail.txt

=================================================================
CLOAKBROWSER PATTERNS
=================================================================

Launch (always use this):
  from cloakbrowser import launch, launch_persistent_context
  ctx = launch_persistent_context(tmpdir, headless=False, humanize=True)
  p = ctx.pages[0] if ctx.pages else ctx.new_page()

Cleanup:
  import atexit, shutil
  td = tempfile.mkdtemp()
  atexit.register(lambda: shutil.rmtree(td, ignore_errors=True))

Key differences from Playwright:
- No executable_path needed (bundled Chromium v146)
- humanize=True for natural mouse/anti-detect timing
- headless=True CRASHES on this system — always use headless=False
- No xvfb-run needed (display server running)
- Stealth is built into the binary (no JS injection needed)

=================================================================
TROUBLESHOOTING
=================================================================

"Missing X server or $DISPLAY":
  - Ensure DISPLAY is exported: export DISPLAY=:1
  - firecrawl_signup.sh sets this automatically

"TargetClosedError":
  - Page/tab was closed (common with PKCE verify links)
  - Scripts now catch this and check for new tabs

Proton not finding email:
  - Email may take 30-60s to arrive
  - Script retries 15 times with 5s waits + inbox reloads
  - Check ~/proton_profile is persisting login (first run needs valid creds)

Checkbox not registering (Clerk/OpenRouter):
  - Use .check(force=True) instead of .click()
  - JS dispatchEvent doesn't update React/Clerk internal state
