# 2026-06-26 Session Notes

## OpenRouter Signup Automation (`openrouter_signup.py`)

### Architecture
- **Steps 2/5/6** (OpenRouter signup, verify, key extraction): Camoufox (anti-fingerprint Firefox)
  - Fresh tmpdir per run via `tempfile.mkdtemp`, cleaned up on exit
  - `persistent_context=True` with `user_data_dir=tmpdir`
- **Step 3** (Proton Mail verification link): Playwright Chromium subprocess
  - Persistent profile at `~/proton_profile` (survives across runs)
  - If profile already has Proton logged in, skips credential entry
  - Runs in subprocess because Camoufox's Firefox engine crashes on Proton Mail JS errors
    - Root cause: Playwright `coreBundle.js` bug — `pageError.location` is undefined
    - Fix: patched coreBundle.js to guard `pageError.location` before accessing `.url`/`.lineNumber`/`.columnNumber`

### Key Technical Details
- **Email generation**: captured from `email.sh` stdout (last line with @), not from `mail.txt`
  - Subprocess env must include: venv `PATH`, `VIRTUAL_ENV`, and `.pythonlibs` in `PYTHONPATH`
  - Without these, `duckmail.py` can't find `camoufox` module
- **Proton inbox search**: uses `/` keyboard shortcut to open search, `keyboard.type()` for email, Enter to search
  - The search input (`data-testid="search-keyword"`) is `readonly` until activated — must use keyboard, not `.fill()`
  - `timeout=0` on "Mail" link click to avoid hanging
- **Proton login skip**: checks if `a:has-text('Mail')` is visible after loading — if yes, skips credentials
- **Individual account**: after verify URL, if on `openrouter.ai`, clicks "Individual" text/button
- **API key extraction** (3 methods):
  1. `<code>` block inner_text + regex `sk-or-v1-[a-zA-Z0-9]+`
  2. "Copy" button + `navigator.clipboard.readText()` + regex
  3. Full HTML regex `sk-or-v1-[a-zA-Z0-9]{20,}`
- **6-digit sign-in code (factor-two)**: NOT handled in script — must be entered manually
- **Retry**: 3 signup attempts; if verification email not found after 5 inbox retries, generates fresh email+password and restarts

### Playwright coreBundle.js Patch
```bash
COREBUNDLE=".../playwright/driver/package/lib/coreBundle.js"
# Guards pageError.location before accessing properties
sed -i 's/url: pageError\.location\.url/url: (pageError.location \&\& pageError.location.url) || ""/g'
sed -i 's/line: pageError\.location\.lineNumber/line: (pageError.location \&\& pageError.location.lineNumber) || 0/g'
sed -i 's/column: pageError\.location\.columnNumber/column: (pageError.location \&\& pageError.location.columnNumber) || 0/g'
```
