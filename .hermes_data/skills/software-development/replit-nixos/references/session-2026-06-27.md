# Session 2026-06-27 — Storage Optimization

## Storage Cleanup Results
- **Before:** 1.8 GB workspace
- **Action:** Symlinked `.cache`, `.pythonlibs`, `.local`, `.config` → ephemeral `~/`
- **After:** ~620 MB workspace (`.hermes_data` 177 MB + `.git` 430 MB + working files)
- **Key insight:** `.git` at 430 MB is most of remaining storage — consider `git gc --aggressive` if it grows further

## Symlink Pattern (Idempotent)
```bash
unset XDG_CACHE_HOME
for d in .cache .pythonlibs .local .config; do
  [ -L "$HOME/workspace/$d" ] && continue
  [ -e "$HOME/$d" ] || mkdir -p "$HOME/$d"
  [ -e "$HOME/workspace/$d" ] && mv "$HOME/workspace/$d"/* "$HOME/$d/" 2>/dev/null
  rm -rf "$HOME/workspace/$d" && ln -sf "$HOME/$d" "$HOME/workspace/$d"
done
```

## Gotcha: `rm -rf "$HOME/$d"` in early versions deleted the real ephemeral dirs
- The old code did `rm -rf "$HOME/$d" && mkdir -p "$HOME/$d"` which wiped ephemeral data on every run
- Fixed to: only mkdir if missing, then migrate + symlink

## hermes.sh Integration
The symlink block is now in `hermes.sh` (line 7-13) so it runs on every setup/reset.
